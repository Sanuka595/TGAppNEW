# Claude Code Instructions (TGPerekup)

## Роль и зона ответственности
Твоя задача (Senior/Architect) — проектирование архитектуры, управление глобальным стейтом React, реализация сложной игровой логики (монополия), настройка и синхронизация Socket.IO, глубокий рефакторинг.
**Права:** Ты используешь MCP-серверы для доступа к файловой системе и git. У тебя настроен bypass на чтение/запись файлов, запуск `build` и `lint`.
**Ограничение:** Перед началом крупного рефакторинга или внедрением новых архитектурных решений, обязательно сформируй пошаговый план (`.md`) и дождись явного утверждения ("ok").

## Язык и коммуникация
* **Общение и планирование:** Все диалоги, roadmap и объяснения архитектуры пиши строго на русском языке.
* **Тональность:** По делу, прямо, без «воды» и извинений. Максимально структурированный Markdown.
* **Код:** Исходный код, JSDoc, переменные и сообщения git-коммитов — строго на английском.

## База знаний (Obsidian)
* Все утвержденные архитектурные решения, схемы состояний стейт-менеджера и документацию по событиям WebSocket обязательно сохраняй в `/docs/obsidian/` в формате Markdown.
* Форматируй эти файлы так, чтобы они легко читались в Obsidian (используй четкую иерархию заголовков, списки, блоки кода).

## Структура монорепозитория
* `/packages/client` — React-фронтенд. Твой фокус: провайдеры, глобальный стейт, оптимизация ререндеров при частых сокет-событиях.
* `/packages/server` — Node.js бекенд. Твой фокус: стейт-машина игры, безопасность обработки событий перекупа.
* `/packages/shared` — Общие контракты. Твой фокус: проектирование строгих интерфейсов для API и DTO для Socket.IO.

## Основные команды (корень проекта)
* Глобальная проверка:
  `npm run typecheck && npm run lint:fix`
* Полная сборка всех пакетов (используй для проверки целостности после рефакторинга):
  `npm run build`
* Прогон интеграционных и unit-тестов:
  `npm run test`

## Специфика проекта и архитектурные конвенции
* **Изоляция:** Поддерживай жесткие границы между пакетами. Клиент и сервер общаются исключительно через интерфейсы из `/shared`.
* **Real-time логика:** При проектировании Socket.IO учитывай потерю соединения на мобильных устройствах (Telegram Web App) — реализуй надежные механизмы реконнекта и восстановления стейта игры.
* **TypeScript:** Strict mode. Полный запрет на `any` и `ts-ignore` без веской причины. Применяй продвинутую типизацию (utility types, generics) для строгой привязки payload'ов к названиям событий сокетов.
