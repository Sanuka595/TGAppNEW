## 1. Zustand Store — структура слайсов

В проекте используется **Zustand 5** с паттерном разделения на слайсы. Основной стор собирается в `gameStore.ts`.

```
packages/client/src/store/
├── gameStore.ts               # Точка сборки, экшены и socket listeners
├── uiStore.ts                 # Стор для UI-состояний (табы, модалки)
└── slices/
    ├── playerSlice.ts         # Баланс, гараж, энергия
    ├── soloSlice.ts           # Логика соло-режима (Бот Боря, квесты)
    └── multiplayerSlice.ts    # Управление комнатами и Socket.IO
```

## 2. Ключевые компоненты (React 19)

### Основной макет
- **App.tsx**: Корневой компонент. Инициализирует сокеты, обрабатывает `start_param` (deep links) и управляет переключением вкладок через `MainLayout`.
- **TopBar.tsx**: Отображает баланс, энергию и репутацию игрока.
- **BottomNavigation.tsx**: Переключатель вкладок (Гараж, Карта, Сделки, Маркет).

### Игровые экраны (game/)
- **RadialBoard.tsx**: Радиальная карта на 12 клеток. Отрисовывается через SVG с анимациями Framer Motion.
- **DiceArea.tsx**: Центральная область карты. Содержит кнопку броска D6 и кнопки тактических ходов (+1, +2, +3).
- **GarageView.tsx**: Список купленных автомобилей игрока с возможностью ремонта и тюнинга.
- **MarketView.tsx**: Витрина доступных для покупки автомобилей.
- **DealsView.tsx**: Биржа P2P займов и текущие сделки.
- **ActionModal.tsx**: Универсальная модалка для взаимодействия с клеткой (покупка, ремонт, аренда).
- **RaceModal.tsx**: Интерфейс проведения гонок и выбора ставок.
- **DevPanel.tsx**: Панель разработчика (только в DEV-режиме) для отладки баланса и перемещений.

## 3. Синхронизация и Socket.IO

Инициализация слушателей происходит в `gameStore.ts` через функцию `initSocketListeners()`.

**Основные события:**
- `room_updated`: Сервер присылает полный `RoomState`. Вызывает `syncRoomState`.
- `dice_roll_result`: Результат броска (значение и новая позиция).
- `sync_action_result`: Результат игрового действия другого игрока.
- `room_error`: Сообщение о неудачной валидации на сервере.

## 4. Обработка Deep Links

Игра поддерживает вход по прямым ссылкам через `start_param` Telegram:
- `solo`: Запуск одиночного режима.
- `multi`: Открытие окна создания/поиска комнаты.
- `reset`: Полный сброс прогресса аккаунта.
- `ROOM_ID`: Автоматическое присоединение к существующей комнате.

## 5. Оптимизация

- **Атомарные селекторы**: `const balance = useGameStore(s => s.player.balance)` предотвращает лишние ререндеры.
- **React.memo**: Используется для `CellIcon` и элементов лога.
- **Decimal.js**: Все финансовые вычисления в сторе производятся только через этот тип, чтобы избежать ошибок окружения в JS.
