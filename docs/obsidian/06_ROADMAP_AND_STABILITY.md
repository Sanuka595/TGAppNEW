# Перекуп D6 — Дорожная карта (Roadmap)

В этом документе описаны приоритетные направления развития проекта для обеспечения стабильности, масштабируемости и вовлеченности игроков.

---

## 1. Стабильность и Надежность (High Priority)

### [ ] Переход на Redis для Room State
- **Проблема**: В текущем состоянии при перезагрузке сервера все игровые комнаты в памяти стираются.
- **Решение**: Использовать Redis для хранения состояния комнат с TTL (например, 24 часа).
- **Результат**: Игроки смогут продолжить игру даже после кратковременного падения или обновления сервера.

### [ ] Механика Full State Sync
- **Проблема**: При потере связи на мобильных устройствах клиент может пропустить важные события `sync_action`.
- **Решение**: Добавить событие `request_full_state`, которое возвращает полный объект `RoomState`. Клиент должен вызывать его автоматически при каждом реконнекте Socket.IO.
- **Результат**: 100% синхронизация состояния независимо от качества интернета.

### [ ] Полное покрытие валидацией (Zod)
- **Проблема**: Некоторые серверные обработчики доверяют данным от клиента.
- **Решение**: Внедрить `schema.parse()` для каждого входящего события в `socketHandlers.ts`.
- **Результат**: Защита от эксплойтов и некорректных состояний.

---

## 2. Масштабируемость архитектуры (Medium Priority)

### [ ] Рефакторинг gameStore.ts
- **Проблема**: Файл разросся до 900+ строк, логика перемешана с состоянием.
- **Решение**: Вынести реализацию экшенов в доменные сервисы или отдельные модули в `shared`. Оставить стор как "тонкое" хранилище.
- **Результат**: Упрощение поддержки и тестирования.

### [ ] Расширение Shared Logic
- **Проблема**: Логика гонок и некоторых расчетов дублируется на клиенте и сервере.
- **Решение**: Перенести все игровые формулы в `packages/shared`.
- **Результат**: Гарантия идентичности расчетов на всех слоях.

---

## 3. Геймплей и Контент (Future)

### [ ] Система достижений и аккаунтов
- **Идея**: Внедрение PostgreSQL для хранения постоянного профиля игрока, его репутации и коллекции редких авто.

### [ ] Процедурная генерация новостей
- **Идея**: Динамическое создание событий на основе активности игроков в комнате (например, если никто не покупает Scrap, запустить "Программу утилизации").

### [ ] Соло-режим 2.0 (Боря-перехватчик)
- **Идея**: Полноценный ИИ-оппонент, который может перекупать машины с рынка прямо перед носом игрока и предлагать кабальные сделки.

---

## 4. Чек-лист проверки качества

- [ ] `npm run typecheck` проходит без ошибок.
- [ ] Все Socket-события имеют JSDoc.
- [ ] Финансовые операции всегда используют `Decimal.js`.
- [ ] Скрытые дефекты (`isHidden`) никогда не передаются в маркет до диагностики (валидация на сервере).
