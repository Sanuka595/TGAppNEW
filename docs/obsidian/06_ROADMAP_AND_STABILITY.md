# Перекуп D6 — Дорожная карта (Roadmap)

В этом документе описаны приоритетные направления развития проекта.

---

## 1. Стабильность и Надёжность (High Priority)

### [ ] Переход на Redis для Room State
- **Проблема**: При перезагрузке сервера все игровые комнаты в памяти стираются.
- **Решение**: Использовать Redis для хранения состояния комнат с TTL (например, 24 часа).
- **Результат**: Игроки смогут продолжить игру даже после кратковременного падения или обновления сервера.

### [ ] Механика Full State Sync
- **Проблема**: При потере связи на мобильных устройствах клиент может пропустить важные события `sync_action`.
- **Решение**: Добавить событие `request_full_state`, возвращающее полный `RoomState`. Клиент должен вызывать его автоматически при каждом реконнекте (событие `connect` уже делает `joinRoom`, осталось добавить явный запрос полного состояния).
- **Результат**: 100% синхронизация независимо от качества интернета.

### [ ] Полное покрытие валидацией (Zod)
- **Проблема**: Некоторые серверные обработчики доверяют данным от клиента.
- **Решение**: Внедрить `schema.parse()` для каждого входящего события в `socketHandlers.ts`.
- **Результат**: Защита от эксплойтов и некорректных состояний.

## 2. Масштабируемость архитектуры (Выполнено ✅)

### [x] Рефакторинг gameStore.ts
- **Было**: Монолитный файл 912 строк — бизнес-логика жёстко перемешана с состоянием Zustand.
- **Сделано**: Декомпозирован на доменные слайсы через Zustand `StateCreator`:
  - `gameStore.ts` → тонкий ассемблер ~68 строк
  - `store.types.ts` → `GameStore` interface (без circular deps)
  - `socketListeners.ts` → Socket.IO listeners в отдельном модуле
  - `slices/playerSlice.ts` → `createPlayerSlice()` со всей логикой игрока
  - `slices/soloSlice.ts` → `createSoloSlice()` с соло-механиками
  - `slices/multiplayerSlice.ts` → `createMultiplayerSlice()` с мультиплеером, гонками, долгами

### [x] Расширение Shared Logic
- **Было**: Логика гонок и константы были продублированы или существовали только на одной стороне.
- **Сделано** (новые экспорты из `packages/shared`):

  **`constants.ts`** — единый источник игровых констант:
  - `DIAGNOSTICS_COST = 200`
  - `MARKET_REFRESH_COST = 500`
  - `ENERGY_PURCHASE_COST = 500`
  - `MAX_ENERGY = 3`
  - `SPECIAL_REPAIR_DISCOUNT = '0.95'`

  **`businessLogic.ts`** — новые чистые функции:
  - `TIER_RACE_BONUS` — перенесён с сервера в shared, оба слоя используют одну константу
  - `SOLO_RACE_TIER_MULTIPLIERS` + `calculateSoloRaceWinChance(car)` — формула вероятности соло-гонки
  - `generateMarketForCell(cellType)` — генерация лотов для клетки (убирает дублирование в `executeCellAction` и `refreshMarket`)
  - `resolveRandomEncounter()` — 15% случайные события на дороге (ГАИ / заначка)

  **Бонус-фикс**: Баг в `refreshMarket` — `type.split('_')[1] as any` давал строку в нижнем регистре (`'bucket'`), что ломало `generateCar`. Исправлено использованием `generateMarketForCell(currentEvent.type)`.
