# Перекуп D6 — Полная техническая документация
## Файл 5 из 5: Сервер, Бот и DevOps

---

## 1. Сервер (Node.js + Socket.IO)

Серверная часть расположена в `packages/server`. Реализована на **Express** и **Socket.IO 4**.

### Ключевые модули:
- **index.ts**: Точка входа. Запускает HTTP-сервер, Socket.IO и встроенного Telegram-бота. Содержит интервал для очистки неактивных комнат (каждые 10 минут).
- **roomManager.ts**: Ядро серверной логики. Управляет созданием комнат, транзакциями (покупка/продажа/ремонт) и валидацией состояний.
- **socketHandlers.ts**: Диспетчер сетевых событий. Принимает запросы от клиентов и вызывает соответствующие методы `roomManager`.

---

## 2. Telegram Bot (node-telegram-bot-api)

Бот интегрирован непосредственно в серверный процесс для экономии ресурсов и упрощения деплоя.

**Функции бота:**
- Обработка команды `/start`.
- Предоставление Inline-кнопок для запуска Mini App в разных режимах (`solo`, `multi`, `reset`).
- Генерация ссылок с `start_param` для входа в конкретные игровые комнаты.

---

## 3. DevOps и Деплой

### Инфраструктура
- **Docker**: Проект контейнеризирован. Dockerfile выполняет сборку всех пакетов монорепозитория и запускает сервер.
- **Railway.app**: Основная площадка для деплоя. Используется автоматическая сборка при пуше в ветку `main`.
- **Healthcheck**: Эндпоинт `/health` возвращает статус сервера и количество активных комнат.

### Переменные окружения
- `TELEGRAM_BOT_TOKEN`: Токен от @BotFather.
- `WEBAPP_URL`: Публичный URL приложения (нужен боту для генерации кнопок).
- `ALLOWED_ORIGINS`: Список разрешенных доменов для CORS.

---

## 4. Безопасность и Валидация

- **Authoritative Server**: Клиент никогда не диктует свой баланс. Все денежные операции проходят через `roomManager` на сервере.
- **Фильтрация Dev-команд**: В `socketHandlers.ts` заблокирована ретрансляция отладочных команд (добавление денег, телепортация) в мультиплеере.
- **Cleanup**: Сервер автоматически удаляет комнаты, в которых не было активности более 30 минут.

---

## 5. Известные проблемы и Технический долг

- **In-memory Storage**: Комнаты хранятся в оперативной памяти. При перезагрузке сервера игровые сессии прерываются (планируется переход на Redis).
- **Init Data Validation**: Требуется внедрение проверки HMAC подписи Telegram Init Data для предотвращения подделки ID игрока.
